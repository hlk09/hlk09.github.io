计算机安全复习

Posted on 2020-01-01 Edited on 2020-01-09

第一讲 概述

考察内容

  1. 计算机安全的概念
  2. 威胁、攻击和资产
  3. 基本安全设计原则

    考察重点

计算机系统资源包含什么

  1. 硬件：计算机系统以及其他数据处理、存储和通信的设备
  2. 软件：操作系统、系统实用程序和应用程序
  3. 数据：文件和数据库，与安全相关的数据，如口令文件等
  4. 通信设施和网络：局域网和广域网的通信链路、网桥、 路由器等

计算机安全的三个目标及其分别的含义

为自动化信息系统提供的保护，以达到保持信息系统资源的机密性(confidentiality)、完整性(integrity)和可用性(availability)的目标。

  1. 数据机密性：确保隐私或机密信息不能由非授权个人利用，或不能披露给非授权个人。
  2. 数据完整性：确保信息和程序只能在指定的和授权的方式下才能够被改变；系统完整性：确保系统在未受损的方式下执行预期的功能，避免对系统进行有意或无意的非授权操作。
  3. 可用性：确保系统能够迅速地进行工作，并且不能拒绝 对授权用户的服务。

威胁的分类及其对应的攻击方式

非授权泄露：实体未授权而获得数据访问 - 破坏机密性

  1. 暴露：直接泄露（故意泄露）
  2. 截获：搭线窃听
  3. 推理：非授权用户推理（通信量）
  4. 入侵：躲避系统安全保护（绕过防火墙，克服系统访问控制保护）

欺骗：导致授权实体接收虚假数据并相信其正确性的情况或事件。- 破坏完整性

  1. 冒充：佯装成授权实体（特洛伊木马）
  2. 伪造：伪造虚假数据
  3. 抵赖：否认对行为的责任（否认发送或接收过数据）

破坏：中断或阻止系统服务和功能正常运行的情况或事件。- 破坏可用性或完整性

  1. 失能：禁用系统组件（物理破坏、系统硬件损害，恶意软件、 特洛伊木马、病毒、蠕虫）
  2. 损坏：修改数据或系统功能（设置后门逻辑）
  3. 阻止系统运行（中断通信链路）

篡夺：导致系统服务或功能被非授权实体控制的事件或情况。-破坏完整性

  1. 盗用：服务窃取（分布式拒绝服务攻击，盗用处理器）
  2. 误用：恶意代码

攻击的定义和分类

定义

攻击：被实施的威胁（威胁行为），如果成功，将会导致 不期望的安全侵害或威胁后果。

分类

  1. 主动攻击：试图改变系统资源或影响其运行
  2. 被动攻击：试图从系统中学习或利用信息，但不影响系统资源
  3. 内部攻击：由安全边界内部的实体（内部人）发起的攻击
  4. 外部攻击：由系统边界外部的非授权用户或非法使用者（外部 人）发起的攻击

考察重点

密码学术语解释

密码学：密码学是关于如何在敌人存在的环境中通讯

密码学发展阶段

  1. 第一阶段：1949年之前，古典密码，密码学是一门艺术
  2. 第二阶段：1949年之后，现代密码出现，密码学成为科学
  3. 第三阶段：1976年以后，公钥密码学诞生

凯撒密码和棋盘密码的原理；字母频率破译法

加密、解密、密码体制、明文、密文、密钥的概念

  1. 加密（encryption）：将消息进行编码，以模糊其含义的过程
  2. 解密（decryption）：是加密的逆过程，将加密过的消息变回其原始形式
  3. 密码体制（cryptosystem）：用于加密和解密的系统
  4. 明文P（plaintext）：消息的初始形式，原始可理解的消息或数据
  5. 密文C（ciphertext）：加密后的形式，不可直观理解，依赖于明文和密钥
  6. 私密的密钥K（key）：用来加密消息的一组秘密的数据（分不同的密码体制）

对称密码概念

加密与解密的密钥相同，P=D(K, E(K,P))

分组密码的两个设计原则及含义

  1. 混乱：在加密变换过程中使明文、密钥和密文之间的关系尽可能复杂化，以防止统计分析。
  2. 扩散：把一位明文的信息扩展到尽可能多的密文中去，以便隐藏明文的统计特性；将每一位密钥的影响也尽可能迅速地扩展到较多的（整个）输出密文中去

DES分组密码算法

  1. 分组加密算法：明文和密文为64位分组长度
  2. 对称算法：加密和解密除密钥编排不同外，使用同一算法
  3. 密钥长度：56位，（64位中每个第8位为奇偶校验位，故64－8＝56位) 长期使用
  4. 设计原则：混乱、扩散
  5. DES抗穷举攻击能力弱，是不安全的
  6. DES算法的替代方案：对DES进行复合，强化抗攻击能力，如三重DES；开辟新方法，既能做到加解密速度快，又具有抗攻击的能力，如AES

消息认证的目的，如何使用对称密码或哈希函数实现消息认证

消息认证的目的

  1. 信源识别，即验证发信人确实不是冒充的
  2. 检验发送信息的完整性, 也就是说, 即使信息确实是经过授权的信源发送者发送的, 也要验证在传送过程中是否被篡改, 重放或延迟。

消息认证的工具

  1. 对称密码体制
  2. 哈希函数

消息认证用对称密码的实现

  1. 发送者和接收者协商秘密的对称密钥K
  2. 发送消息时，发送 （m，C），C= E(m, K)
  3. 接收者收到（m，C）后，计算E(m, K)，与C对比，若相同，成功；否则，失败
  4. 失败原因：m或C被篡改

消息认证用哈希函数的实现

利用Hash函数生成消息的一个固定长度码字，称为消息认证码（MAC），级联在消息后面发送

  1. 发送者和接收者协商秘密的串K
  2. 发送消息时，发送（m，C），C= H(m,K)
  3. 接收者收到（m，C）后，计算H(m,K)，与C对比，若相同，成功；否则，失败

哈希函数的定义，要求，应用举例

哈希函数定义

是一种典型的消息认证函数，提供消息的完整性保护

哈希函数要求

  1. 对于任意给定的x，计算H(x)较容易，用硬件、软件都可实现
  2. 单向性（抗原象性）：对于任意给定的散列值h，找到满足H(x)=h的x是不可行的
  3. 抗若碰撞性（第二抗原像性）：给定一个x，找到一个y，使得y≠x，且H(y)=H(x)，是计算上不可行的
  4. 抗强碰撞性：找到任何满足H(y)=H(x)的一对(x,y)在计算上不可行

应用举例：

  1. 消息认证
  2. 口令：操作系统中存储的口令，QQ用户口令等，都是用 Hash值来存储。比如，口令是x，实际存储的是 H(x)。若H(x)被攻击者窃取，无法恢复x；利用哈希函数的单向性。
  3. 入侵检测：对电脑中的文件F，计算H(F)，并秘密保存。可以通 过重新计算H(F)，来检测文件是否被修改。

第四讲 访问控制

考察内容

考察重点

访问控制机制中的主体、客体、访问权概念

  1. 主体：能够访问客体的实体
  2. 客体：外界对其访问受到控制的资源
  3. 访问权：描述主体可以访问客体的方式

访问控制的目的，实现的安全策略

  1. 目的：防止未经授权使用资源，包括防止以非授权方式使用资源，从而使计算机系统在合法范围内使用。
  2. 策略：对于每个具体的系统资源，谁或什么可以访问，以及每个实例允许的访问类型

访问控制机制分类及其分别的含义

  1. 自主访问控制–DAC：每个主体拥有一个用户名并属于一个组或具有一个角色，每个客体都拥有一个限定主体对其访问权限的访问控制列表，通过访问控制列表（或授权列表）来限定哪些主体针对哪些客体可以执行什么操作
  2. 强制访问控制–MAC：主体的权限和客体的安全属性都是固定的，由系统决定主体是否可以访问客体。不允许主体干涉访问控制。
  3. 基于角色的访问控制：基于角色的访问控制是一个复合的规则。一个身份被分配给一个被授权的组。起源于UNIX系统或别的操作系统中组的概念。

访问控制列表、访问能力表、访问控制矩阵

  1. 访问控制矩阵：按列看是访问控制表内容，按行看是访问能力表内容。
  2. 访问控制列表：以客体为中心，每个客体附加一个可以访问它的主体的明细表。
  3. 访问能力列表：每个主体都附加一个该主体可以访问的客体的明细表。

第五讲 入侵检测（Intrusion Detection System，简称IDS）

考查重点

入侵者概念

安全入侵和入侵检测定义

  1. 安全入侵：一个安全事件或多个安全事件的组合构成一个安全事故（security incident），在安全事故中入侵者在未经授权的情况下获取或尝试获取一个系统（或系统资源）的访问权。
  2. 入侵检测：一种监控并分析系统事件的安全服务，目标是发现未经授权而访问系统资源的尝试活动，并提供实时或近似实时的报警。

入侵检测系统组件

  1. 传感器：负责收集数据
  2. 分析器：从一个或多个传感器或其他分析器接收输入，负责确定是否发生了入侵，提供指导，用于判定什么活动是入侵导致的后果
  3. 用户接口：用户查看系统输出

审计记录概念、格式

  1. 概念：入侵检测系统的输入，用户正在进行的活动的记录
  2. 格式：主体、动作、客体、异常条件、资源使用、时间戳

入侵检测两种方法及优缺点：基于特征或基于异常

异常（行为）检测:

  1. 定义：依据合法用户的行为规则来判定用户行为的不合法性
  2. 优缺点：

特征检测

  1. 定义：根据一组规则或攻击模式，来确定给定的行为是否为入侵者行为
  2. 优缺点：

掌握入侵检测系统的分类及其各自概念

  1. 基于主机的入侵检测系统（HIDS）：运行在单个工作站、客户端或主机上，检测的是主机上的用户和软件活动
  2. 分布式基于主机的入侵检测系统（DIDS）：主机代理模块、局域网监视代理模块、中心管理员模块
  3. 基于网络的入侵检测系统（NIDS）：是附加在网络上的一台单独的设备，从网络传输流中获取网络包，监测特定的网段或设备的流量并分析网络、传输和应用协议，用以识别可疑的活动
  4. 分布式或混合式入侵检测系统：每个主机和网络设备都是潜在传感器，提高更高覆盖率

蜜罐/蜜网的概念和及其功能

  1. 蜜罐计算机：是一台不作任何安全防范措施而且连接网络的计算机，管理员能够控制。安装了敏感的监控器和事件记录器，检测和收集攻击者活动信息。
  2. 蜜罐网络: 它是一个专门设计来让人“攻陷”的网络，一旦被入侵者攻破，入侵者的一切信息、工具等都将被用来分析学习

第六讲 防火墙

考查重点

防火墙概念、三个设计目标、功能

防火墙概念

防火墙（Firewall）是在两个网络之间执行访问控制策略的安全系统。它是一种计算机硬件和软件系统集合，是实现网络安全策略的有效工具之一，被广泛地应用到内部网络与外部网络的边界位置。

三个设计目标

  1. 所有入站和出站的网络流量都必须通过防火墙
  2. 只有经过授权的网络流量才允许通过防火墙
  3. 防火墙本身不能被渗透，应该运行在有安全操作系统的可信系统上

功能

  1. 保护脆弱和有缺陷的网络服务
  2. 实施安全策略，加强对网络系统的访问控制
  3. 防止内网信息暴露，加强隐私
  4. 对内外网之间的通信进行监控审计

防火墙分类

从防火墙技术分

  1. 包过滤FW
  2. 状态检测FW
  3. 应用级网关FW
  4. 电路级网关FW

从软硬件形式分

  1. 硬件FW
  2. 软件FW

包过滤防火墙工作原理

将数据包的包头信息（如源IP地址、目的IP地址、源端口号、目的端口号、协议类型、方向等）与事先设定的过滤规则相比较，据此决定转发或者丢弃该包

按TCP包头信息建立过滤规则

状态检测防火墙的工作原理

动态包过滤，工作在网络层。它将属于同一连接的所有包作为一个整体的数据流看待（保证了前后报文的相关性），构成连接状态表。通过规则表与状态表的共同配合，对数据包进行转发或拒绝处理。

代理服务器工作原理，功能、特点

原理

网络信息的中转站

功能

  1. 设置用户验证和记账功能，可按用户进行记账，没有登记的用户无权通过代理服务器访问Internet网
  2. ）对用户进行分级管理，设置不同用户的访问权限，对外界或内部的Internet地址进行过滤，设置不同的访问权限。
  3. 连接内网与Internet，充当防火墙（Firewall），节省IP开销
  4. 增加缓冲器（Cache)，提高访问速度，对经常访问的地址创建缓冲区，大大提高热门站点的访问效率。

优点

  1. 代理服务器理解应用协议，它针对特定的应用协议使用指定的数据过滤逻辑。根据协议内部可见的内容而不仅仅根据头部数据来控制穿越防火墙的行为；
  2. 安全性高，可防止在受信主机与非受信主机间直接建立联系。

缺点

  1. 每个特定的应用（www、FTP、E-mail）都需要一个专门的代理，扩充性差，使用时工作量大，效率低，在网络上运行一个代理服务器可能需要更昂贵的硬件；
  2. 首先它会使访问速度变慢，因为不允许用户直接访问网络。

软、硬件防火墙概念

  1. 软件防火墙：运行在通用计算机上的纯软件，安装在 公共的操作系统上，简单易用，配置灵活；但因底层操作系统是一个通用型的系统，其数据处理能力、安全性能水平都比较低。
  2. 硬件防火墙：将防火墙软件固化在专门设计的硬件平台上（防火墙程 序做到芯片），数据处理能力与安全性能水平都得到了很大的提高。但因来自网络的威胁不断变化，防火墙的安全策略、配置等也需要经常进行调整，而纯硬件防火墙的调整非常困难。一般应用于性能要求高、安全策略比较稳定的情况

防火墙4种部署方式

  1. 屏蔽路由器
  2. 堡垒主机
  3. 屏蔽主机结构
  4. 屏蔽子网结构

第七讲 恶意软件

考查重点

恶意软件的概念、分类

恶意软件：一种被植入系统中，以损害受害者数据、应用程序或操作系统的可信性、完整性或可用性，抑或对用户实施骚扰或妨碍的程序

  1. 按是否依附于宿主程序：寄生程序：依附于宿主程序的程序片段，如病毒；非寄生程序：独立于宿主程序，可被操作系统调度和执行的独立程序，如蠕虫、Bot程序、木马
  2. 按是否进行复制：不进行复制：触发执行的程序或程序片段，如后门、Bot程序；进行复制：程序或程序片段，执行时生成一个或多个自身的拷贝，在合适的时机在当前系统或其他系统中被激活，如病毒、蠕虫

后门、逻辑炸弹

  1. 后门：能够绕过正常安全检查的任何机制，可允许未经授权访问某些功能
  2. 逻辑炸弹：被入侵者插入到正常软件中的程序，但预定义的条件满足时，逻辑炸弹被触发，开始执行非授权操作，其他时间处于休眠状态

第八讲 数据库安全

考查重点

数据库管理系统的管理策略

  1. 集中管理：少量特权用户可以授予和回收访问权
  2. 基于所有权的管理：表的所有者（创建者）可以授予和回收该表的访问权
  3. 分散管理：表的所有者除了可以授权和回收该表的访问权外，还可以对其他用户授予和回收授权权，以允许他们对该表授予和回收访问权（级联授权）

数据库访问控制和操作系统访问控制的区别

  1. 控制粒度：操作系统中控制的粒度是文件，数据库中则需要控制到记录和域一级
  2. 关联性：操作系统中的文件之间没有关联关系；数据库中，不仅库表文件之间有关联，在库表内部记录、域、元素都是相互关联的
  3. 若要限制推理则意味着要禁止一些访问路径，即限制访问，但也限制了某些用户正常查询；而操作系统中不要担心推理攻击

推理攻击、推理信道、元数据定义

  1. 推理攻击：完成授权查询并从得到的合法响应中推导出非授权信息的过程
  2. 推理通道：获得非授权数据的信息传送路径
  3. 元数据: 有关数据项之间相关性或依赖性的知识性

特征式、查询集定义

抗击推理攻击的两种方式

  1. 数据库设计时的推理检测：修改数据库结构或者改变访问控制机制以消除推理通道
  2. 查询时的推理检测：在一个查询或多个查询执行期间消除推理通道，如果发现推理通道，查询被拒绝

查询规模限制、查询重叠集控制概念

数据扰动、输出扰动概念

如何访问加密数据库

  1. 数据库中的每一项属性值都被加密
  2. 当用户询问该加密数据库时，修改查询语言为加密后的
  3. 当用户询问该加密数据库时，系统从加密数据库中返回结果
  4. 服务器将上述结果进行解密

第九讲 拒绝服务攻击

考查重点

拒绝服务攻击与分布式拒绝服务攻击概念

  1. DoS：一种通过耗尽CPU、内存、带宽以及磁盘空间等系统资源，来阻止或削弱对网络、系统或应用程序的授权使用的行为
  2. DDoS：如果处于不同位置的多个攻击者同时向一个或多个目标发起拒绝服务攻击,或者一个或多个攻击者控制了位于不同位置的多台机器并利用这些机器对受害者同时实施拒绝服务攻击

拒绝服务攻击对象

  1. 网络带宽
  2. 系统资源
  3. 应用资源

剧毒型、风暴型DoS的攻击机制

剧毒包型DoS：Land攻击、Ping of death 攻击、循环攻击、 WinNuke攻击

  1. 泪滴(Teardrop)攻击：利用异常的数据分片导致接收方在处理分片数据时崩溃，也称为泪滴(Teardrop) 攻击
  2. Land攻击：原是一段C程序，其功能是向受害者发送TCP SYN包，而这些包的源IP地址和目的IP地址被伪造成受害者的IP地址，源端口和目的端口也是相同的（端口必须是激活的？），目标系统在收到这样的包以后可能会挂起、崩溃或重启。
  3. Ping of death攻击：利用协议实现漏洞，向受害者发送超长的Ping数据包，导致受害者系统异常
  4. 循环攻击：当两个都会产生输出的端口(可以是一个系统／一台机器的两个端口，也可以是不同系统／机器的两个端口) 之间建立连接以后，第一个端口的输出成为第二个端口的输入，导致第二个端口产生输出，同时第二个端口的输出又成为第一个端口的输入，如此，一两个端口间将会有大量的数据包产生，导致拒绝服务
  5. WinNuke攻击：

风暴型DoS：ICMP/TCP SYN/UDP洪泛 /Smurf/NTP monlist

  1. TCP洪流(Floods)。向目标主机发送大量设置了不同标志的TCP分组。常被利用的标志包括：SYN, ACK, RST。其中，TCP SYN攻击导致目标主机不断地为TCP连接 配内存，从而使其它功能不能分配到足够的内存。Trinoo就是一种分布式的TCP SYN DoS攻击工具。
  2. ICMP Echo 请求/响应报文(如Ping Floods)。向目标主机发送大量的ICMP分组。
  3. UDP洪流。向目标主机连续发送大量较长的UDP数据包，占用网络带宽，达到阻塞网络的目的。
  4. SYN Flood：发送大量SYN报文，但对服务器的SYN＋ACK应答报文不作应答，即三次握手的第三次握手无法完成，造成服务器维护大量的半连接列表，消耗服务器半连接资源（一般系统的上限为1024，超过此数则不接受新的连接请求）的攻击方式。
  5. Smurf攻击：
  6. NTP攻击

DoS检测和应对

检测

  1. 根据DDoS攻击工具的特征标志检测
  2. 根据异常流量检测

应对

  1. 通过丢弃恶意分组的方法保护网络
  2. 在源端控制DDoS攻击
  3. 追溯 (Traceback) 攻击的源端, 然后阻止它发起新的攻击
  4. 路由器动态检测流量并进行控制

  1. TCSEC标准是计算机系统安全评估的第一个正式标准，具有划时代的意义。该准则于1970年由美国国防科学委员会提出，并于1985年12月由美国国防部公布。TCSEC最初只是军用标准，后来延至民用领域。TCSEC将计算机系统的安全划分为4个等级、7个级别。（D最小）
